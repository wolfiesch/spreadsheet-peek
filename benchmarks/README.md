# Benchmarks

Token-cost measurements for the claims in the main README.

## Methodology

We measure the token cost of xleak's output modes using `tiktoken`'s `cl100k_base` encoding (the GPT-4 tokenizer). This is a proxy for Claude's tokenizer - the absolute numbers differ slightly, but the **ratios between modes are stable** across tokenizers because they're driven by the same structural properties (how many Unicode box-drawing chars, how much repeated whitespace).

All measurements are against [`examples/sample-financials.xlsx`](../examples/sample-financials.xlsx), a synthetic multi-sheet financial workbook generated by [`examples/generate_sample.py`](../examples/generate_sample.py). Reproduce with:

```bash
uv run --with tiktoken --with openpyxl python benchmarks/measure_tokens.py
```

## Results

Last measured: 2026-04-16 · xleak 0.2.5 · tiktoken `cl100k_base`

| Mode | Tokens | Bytes | Data rows | Tokens/row |
|------|-------:|------:|----------:|-----------:|
| Box-drawing (5 rows)   | 593   | 2,888 | 5  | 118.6 |
| Box-drawing (15 rows)  | 1,333 | 6,831 | 15 | 88.9  |
| Text export (head -5)  | 117   | 255   | 5  | 23.4  |
| Text export (head -15) | 328   | 702   | 15 | 21.9  |
| CSV export (head -5)   | 119   | 285   | 5  | 23.8  |

**Key ratio**: Box-drawing costs ~5.1x more tokens per row than text export at 5 rows (~4.1x at 15 rows as fixed overhead amortizes).

## Why these numbers matter

An agent that previews 10 spreadsheets during a session at 15 rows each:

- **Box-drawing everywhere**: 10 × 1,333 = **13,330 tokens** just on previews
- **Box-drawing first, text export after**: 1,333 + (9 × 328) = **4,285 tokens**

That's a **9,000-token savings** - roughly 5% of a 200K context window, preserved for actual work. The savings compound over long debugging sessions.

## What's not measured (yet)

- **Claude tokenizer ground truth**: Anthropic's tokenizer isn't publicly available, so we use cl100k_base as a proxy. PRs welcome if Anthropic publishes a client-side tokenizer.
- **Generation-side token cost**: The box-drawing output is what the *agent reads* (context tokens). The *generation cost* of writing `xleak file -n 15` vs writing a 20-line Python script is a separate measurement - we estimate ~150-200 tokens saved per invocation on the generation side.
- **Real-world workbook variance**: Wider tables (30+ columns) amplify box-drawing overhead; narrower tables (3-5 columns) reduce it. Our sample has 7 columns, which is typical for financial data.

## Contributing benchmark data

If you have an interesting workbook shape (very wide, very tall, CSV instead of xlsx), open a PR adding it to `examples/` and rerun `measure_tokens.py` - the results table here should grow over time.
