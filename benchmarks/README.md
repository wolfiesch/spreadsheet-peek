# Benchmarks

Token-cost measurements for the claims in the main README.

## Methodology

We measure the token cost of xleak's output modes using `tiktoken`'s `cl100k_base` encoding (the GPT-4 tokenizer). This is a proxy for Claude's tokenizer - the absolute numbers differ slightly, but the **ratios between modes are stable** across tokenizers because they're driven by the same structural properties (how many Unicode box-drawing chars, how much repeated whitespace).

Measurements cover two sample workbooks at opposite ends of the shape spectrum:

- [`examples/sample-financials.xlsx`](../examples/sample-financials.xlsx) - 7 columns, typical financial-report shape. Generated by [`examples/generate_sample.py`](../examples/generate_sample.py).
- [`examples/wide-table.xlsx`](../examples/wide-table.xlsx) - 29 columns, an operations-dashboard stress test. Generated by [`examples/generate_wide_table.py`](../examples/generate_wide_table.py).

Reproduce with:

```bash
uv run --with tiktoken --with openpyxl python benchmarks/measure_tokens.py
```

## Results

Last measured: 2026-04-16 · xleak 0.2.5 · tiktoken `cl100k_base`

| Mode | Tokens | Bytes | Data rows | Tokens/row |
|------|-------:|------:|----------:|-----------:|
| Financials (7 cols) - Box-drawing (5 rows)   | 593   | 2,888  | 5  | 118.6 |
| Financials (7 cols) - Box-drawing (15 rows)  | 1,333 | 6,831  | 15 | 88.9  |
| Financials (7 cols) - Text export (head -5)  | 117   | 255    | 5  | 23.4  |
| Financials (7 cols) - Text export (head -15) | 328   | 702    | 15 | 21.9  |
| Financials (7 cols) - CSV export (head -5)   | 119   | 285    | 5  | 23.8  |
| Wide (29 cols) - Box-drawing (5 rows)        | 2,263 | 13,558 | 5  | 452.6 |
| Wide (29 cols) - Box-drawing (15 rows)       | 5,637 | 33,287 | 15 | 375.8 |
| Wide (29 cols) - Text export (head -5)       | 632   | 1,255  | 5  | 126.4 |
| Wide (29 cols) - Text export (head -15)      | 1,854 | 3,627  | 15 | 123.6 |
| Wide (29 cols) - CSV export (head -5)        | 626   | 1,463  | 5  | 125.2 |

**Key ratios**:
- Financials: box-drawing costs ~5.1x more tokens per row than text export (5-row), ~4.1x at 15 rows.
- Wide: box-drawing costs ~3.6x per row (5-row), ~3.0x at 15 rows. The *ratio* is smaller because text-export baseline is larger per row; the *absolute* per-row savings (326 tokens vs 95 for financials) is ~3.4x higher.

## Why these numbers matter

An agent that previews 10 spreadsheets during a session at 15 rows each, assuming a mix:

- **Box-drawing everywhere (financials-shaped)**: 10 × 1,333 = **13,330 tokens** just on previews.
- **Box-drawing everywhere (wide-shaped)**: 10 × 5,637 = **56,370 tokens** - roughly 28% of a 200K window on previews alone.
- **Mode switch (box first, text after)**: 1,333 + (9 × 328) = **4,285 tokens** on the financials path; 5,637 + (9 × 1,854) = **22,323 tokens** on the wide path.

The wide-table case is where this matters most in absolute terms: a single naive 15-row preview already costs more than four financial-shape previews. On a workbook wider than ~25 columns, skip box-drawing entirely and go straight to `--export text | head`.

## What's not measured (yet)

- **Claude tokenizer ground truth**: Anthropic's tokenizer isn't publicly available, so we use cl100k_base as a proxy. PRs welcome if Anthropic publishes a client-side tokenizer.
- **Generation-side token cost**: The box-drawing output is what the *agent reads* (context tokens). The *generation cost* of writing `xleak file -n 15` vs writing a 20-line Python script is a separate measurement - we estimate ~150-200 tokens saved per invocation on the generation side.
- **CSV path**: `xleak` doesn't read CSV files; the SKILL.md CSV fallback uses `head`/`column`/`csvlook` instead. Benchmarking those is straightforward but the variance between tools (and CSV shapes) makes a single table misleading - left as a TODO.

## Contributing benchmark data

If you have an interesting workbook shape (very wide, very tall, CSV instead of xlsx), open a PR adding it to `examples/` and rerun `measure_tokens.py` - the results table here should grow over time.
